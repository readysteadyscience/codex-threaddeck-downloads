# ThreadDeck v0.2.1 - Default Trigger Diagnostics

ThreadDeck v0.2.1 is a distribution refresh for the v0.2 control-plane preview. It keeps the source code private and updates the public downloads package with the default-trigger and runtime diagnostics fixes needed for normal Codex project prompts to route through CTD more reliably.

## Changed

- Broadens the ThreadDeck Skill frontmatter so Codex can select CTD for ordinary installed-project prompts, not only explicit dispatch/control requests.
- Adds runtime doctor reporting for `Skill default trigger: ready` vs `too narrow`.
- Adds a skill-trigger copy gate so future packages cannot regress to a narrow activation description.
- Includes the reviewed user-level hook installer/rollback helper in the plugin preview package.
- Includes the hidden CTD Home history-vault preview for local history references and read-only retrieval.
- Keeps `Launch CTD` as compatibility/debug recovery only; it is not the normal user entry path.

## Boundaries

- The plugin preview does not inject Codex thread tools into arbitrary conversations.
- Hook output remains advisory and does not rewrite prompts, create threads, or dispatch messages.
- Visible worker creation, high-risk actions, publish/release/deploy/account/credential/destructive operations still require explicit confirmation and available Codex tools.
- The public downloads repository provides installable packages and update metadata, not source code.

## Upgrade Notes

Use `latest.json` or `channels/stable.json` to locate the current packages. Existing v0.2.0 artifacts remain available for reproducibility, while stable now points to v0.2.1.
